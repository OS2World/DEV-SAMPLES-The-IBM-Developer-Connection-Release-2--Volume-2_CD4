IBM's VisualAge for Java, Bean Editor Deluxe is much more than a powerful two-way editor. In addition to aiding in the development of Java applets and applications, the VisualAge for Java Visual Composition Editor is a powerful application development tool for rapidly building JavaBean components and for assembling applications composed of JavaBeans. Because VisualAge for Java is based on JDK 1.1 and is JavaBean aware, larger portions of an application can be generated with the VisualAge Visual Composition Editor without the need of adding and modifying code with a text editor. New and existing Java classes can be packaged into JavaBean JAR files and integrated into the Visual Composition Editor with relative ease.

The article introduces readers to the benefits of basing the Visual Composition Editor on a JavaBean infrastructure and to demonstrate the mechanics of integrating new and existing JavaBeans into the tool. The Visual Composition Editor uses JavaBeans in three ways:

  • Allows third-party JavaBeans to be imported into the VisualAge workspace and added to the Visual Composition Editor's palette.
  • Allows developers to assemble, manipulate, and connect JavaBeans to create applets, applications, and new JavaBean classes.
  • Generates JavaBean conforming components and source code.
