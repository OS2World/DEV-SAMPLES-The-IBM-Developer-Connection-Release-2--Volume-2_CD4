CICS Client for Windows NT V2.0.4 (P5)

This product is located on Premier Disc 5.

CICS Clients provide the basis for IBM's very successful Application Mining initiative, which enables customers to bring significant value to new desktop applications by tapping the hidden assets which exist in their core enterprise applications.

CICS Clients provide an easy and low-cost mechanism to enable access to CICS applications for users at desktop workstations or on the Internet. They may be used with CICS servers on all platforms, including IBM Transaction Servers and IBM TXSeries™.

What's New

  • Telnet TN3270 support has been added to the CICS Clients for OS/2®, Windows 95, and Windows NT. This allows multiple Telnet 3270 clients to connect to CICS servers via a CICS Client. It is supported with all CICS servers which support EPI. With the CICS Clients for Windows 95 and Windows NT, code page conversion is supported only in US English; with the CICS Client for OS/2 code page conversion is supported with all supported code pages, including DBCS code pages.

  • Support for NetWare for SAA, previously only available in the CICS Clients for DOS and Windows 3.1, has been added to the CICS Clients for Windows 95 and Windows NT. The support with these two platforms is for SBCS code pages only, and NetWare for SAA on these platforms may not be used with clients running DBCS code pages.

  • SNA LU alias name support has been added to the CICS Clients for Windows 3.1, Windows 95, Windows NT, and OS/2. When used with the appropriate products, this allows LU names to be specified as aliases instead of as fully qualified network names.

  • The CICS Clients for OS/2, Windows 95 and Windows NT support concurrent ECI and EPI sessions from the same application.

  • Support has been provided in the terminal emulator in the CICS Clients for Windows 3.1, Windows 95 and Windows NT for a wider range of key remappings than previously available, adding Scroll Lock and Pause to the keys which may be mapped to any selected function.

  • System Management ECI and EPI exits are supported on the CICS Clients for OS/2, Windows 3.1, Windows 95 and Windows NT. (These exits are similar to those previously provided in the CICS for UNIX Clients).

  • API Tracing has been added to the CICS Clients for OS/2, Windows 3.1, Windows 95 and Windows NT, from Version 2.0.3. When activated, this enables a trace of ECI and EPI calls and their return codes.

  • The CICS Clients for Windows 95 and Windows NT support Windows 3.1 (that is, 16-bit) applications (except for 16-bit OO applications).

  • The DCE RPC communication support in the CICS Client for Windows NT Version 2.0.2, and later, may now be used with the IBM TXSeries, Version 4.2, servers for AIX®, HP-UX, Windows NT, and Solaris.

  • The CICS Clients Version 2.0.4 for OS/2, Windows 3.1, Windows 95, and Windows NT incorporate enhancements to password management. These provide security pop-ups giving appropriate prompts in situations such as wrong password, unknown userid, expired password, or no password entered. This function is only available with servers which support the PEM (Password Expiry Management) service transaction: at present this is CICS Transaction Server for OS/390 Release 1, or later; CICS/ESA Version 4.1; or CICS/ESA Version 3.3 with PTF UN90057. For the protocol drivers with which this function is supported, see the Software Requirements section.

Software Requirements

  • For use of Netware for SAA with the CICS Clients for Windows 95 or Windows NT, NetWare for SAA Version 2.2, with PTF SAA22010, is required in the Gateway server. The NetWare for SAA client requires APAR JR10793 applied.

  • With the CICS Clients for Windows 95 and Windows NT, for SNA communications as well as TCP62, IBM Personal Communications AS/400 and 3270, Version 4.2 is required, with APAR IC20025.

  • Products with which LU alias names may be used are:
    • IBM Personal Communications
    • Microsoft SNA Server - client
    • Microsoft SNA Server
    • IBM Communications Server for Windows NT - client
    • IBM Communications Server for Windows NT
    • Netware for SAA Client (Note that with NetWare for SAA, LU names can only be aliases)

  • Password management using the PEM service transaction is supported using the following protocol drivers:
    • Client platform
      • TCP62:
        • CCLTCP62
      • SNA:
        • CCLWNTSN
