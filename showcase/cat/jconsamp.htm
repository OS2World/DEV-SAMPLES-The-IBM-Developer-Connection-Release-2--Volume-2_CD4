VisualAge for Java and Java Connectors Workshop Samples (M4) (N)

This product is located on Member Disc 4.

The sample code in these subdirectories has been written to supplement a VisualAge for Java and Java Connectors workshop and seminar. The following subdirectories contain:

  • CGI-BIN: CGI-BIN version of AppsGuide.exe and a few smaller samples. AppsGuide.exe uses DB2 stored procedures written in C (stored procedures can be found below)
  • CORBA: CORBA version of AppsGuide.java and a HelloWorld sample.
  • DBCALLS: Contains DB2 stored procedures.
  • DB_BACKUP: Contains backup of APPSGUID DB2 database used in samples.
  • EAB: Contain custom DB2 enterprise access beans (ISVAppsMgr.jar and ISVAppsMgrRMI.jar)
  • JDBC: Contains Java Stored Procedures and utilites to register them.
  • RMI: Contains RMI version of AppsGuide.java, a HelloWorld sample, and a simple "Hotel Management System" framework built using RMI and JDBC calls.
  • SERVLETS: Sample servlets, an AppsGuide.java version which uses servlets, and a San Francisco sample that uses servets.
  • VISUALAGE: Several versions of the COM.ibm.sdm.appsguide package that can be imported into the VisualAge for Java repository.

Software Requirements

  • Windows NT Server 4.0
  • Any JDK1.1 enabled Web browser, such as:
    • Netscape Communicator 4.05 or earlier version with JDK 1.1 AWT patch, or
    • Microsoft Internet Explorer 4.0
  • Universal DB2
  • Servlet Express
  • Any Web Server that Servlet Express supports (eg., Domino Go Webserver 4.6)
  • Any C/C++ compiler (for CGI-BIN demos)
  • VisualAge for Java 1.0 or higher, Enterprise Edition
Utilities
  • idltojava.exe
  • nmake.exe (only need if you wish to use the supplied MAKE files)
SDKs
  • JDK1.1.4 or higher
  • JDK1.2Beta2 or higher (http://java.sun.com)

Installation Instructions

To Install from the CD:

  • Copy the files in the X:\jconsamp directory to the \jconsamp directory on your hard drive. (X represents your CD-ROM drive.)
  • Refer to the \jconsamp\classpath.txt file for information on how to specify your classpath.
