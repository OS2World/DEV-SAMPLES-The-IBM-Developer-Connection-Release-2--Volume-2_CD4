CICS Client for OS/2 V2.0.4 (P1)

This product is located on Premier Disc 1.

CICS Clients provide the basis for IBM's very successful Application Mining initiative, which enables customers to bring significant value to new desktop applications by tapping the hidden assets which exist in their core enterprise applications.

CICS Clients provide an easy and low-cost mechanism to enable access to CICS applications for users at desktop workstations or on the Internet. They may be used with CICS servers on all platforms, including IBM Transaction Servers and IBM TXSeries™.

What's New

  • Telnet TN3270 support has been added to the CICS Clients for OS/2 ®, Windows 95, and Windows NT. This allows multiple Telnet 3270 clients to connect to CICS servers via a CICS Client. It is supported with all CICS servers which support EPI. With the CICS Clients for Windows 95 and Windows NT, code page conversion is supported only in US English; with the CICS Client for OS/2 code page conversion is supported with all supported code pages, including DBCS code pages. This function was introduced in the CICS Clients Version 2.0.3.

  • SNA LU alias name support has been added to CICS Clients for Windows 3.1, Windows 95, Windows NT, and OS/2. When used with the appropriate products, this allows LU names to be specified as aliases instead of as fully qualified network names.

  • The CICS Clients for OS/2, Windows 95 and Windows NT support concurrent ECI and EPI sessions from the same application.

  • CICS Host TCP/IP Access Feature (TCP62) support is added to the CICS Client for OS/2, enabling it to communicate using a TCP/IP network with either CICS/ESA™ Version 4.1, or CICS Transaction Server for OS/390™ Version 1.1, or later.

  • System Management ECI and EPI exits are supported on the CICS Clients for OS/2, Windows 3.1, Windows 95 and Windows NT, as from Version 2.0.3. (These exits are similar to those previously provided in the CICS for UNIX Clients).

  • API Tracing has been added to the CICS Clients for OS/2, Windows 3.1, Windows 95 and Windows NT, from Version 2.0.3. When activated, this enables a trace of ECI and EPI calls and their return codes.

  • The CICS Clients Version 2.0.4 for OS/2, Windows 3.1, Windows 95, and Windows NT incorporate enhancements to password management. These provide security pop-ups giving appropriate prompts in situations such as wrong password, unknown userid, expired password, or no password entered. This function is only available with servers which support the PEM (Password Expiry Management) service transaction: at present this is CICS Transaction Server for OS/390 Release 1, or later; CICS/ESA Version 4.1; or CICS/ESA Version 3.3 with PTF UN90057.

  • Modifications have been made to the CICS Client for OS/2 Version 2.0.4 SNA driver function with IBM Communications Server. CRSR transaction program definition is now done by the CICS Client (and will automatically overwrite any previous definition).

Software Requirements

  • SNA communications support for the CICS Client for OS/2 Version 2.0.4 requires IBM Communications Server for OS/2 Warp, Version 4.0, or later (including IBM eNetwork Communications Server for OS/2 Warp, Version 5.0).

  • TCP62 support for the CICS Client for OS/2 requires IBM eNetwork Communications Server for OS/2 Warp, Version 5.0; APAR JR10793 must be applied.

  • Products with which LU alias names may be used are:
    • IBM Communications Manager/2
    • IBM Communications Server for OS/2 Warp Version 4 X

  • Password management using the PEM service transaction is supported using the following protocol drivers:
    • Client platform
      • TCP62:
        • CCLTCP62
      • SNA:
        • CCLIBMSN
